Gecko is a high-performance Order Management System (OMS) designed for high-frequency trading environments. It handles the entire lifecycle of orders from creation to execution, with sophisticated state management and error handling capabilities.
Features



Comprehensive Order Lifecycle Management: Tracks orders from creation through execution and settlement

Sophisticated State Machine: Ensures proper state transitions with validation

Event-Driven Architecture: Uses NATS for decoupled, highly available communication

Error Recovery: Robust error handling with automatic retries and fallback mechanisms

Monitoring: Timeout detection and state tracking

Architecture

Gecko uses a modern event-driven architecture built on:


Rust: For performance, safety, and reliability

NATS: High-performance messaging system for component communication

Tokio: Asynchronous runtime for concurrent operations

Serde: Serialization/deserialization for message formats

For detailed architecture information, see Architecture Documentation.
Getting Started

Prerequisites


Rust 1.70 or later
NATS server 2.9 or later
Cargo and rustup

Installation



Clone the repository

git clone https://github.com/yourusername/gecko.git
cd gecko




Build the project

cargo build --release




Configuration

Gecko uses environment variables for configuration:

OMS_ERROR_TOPIC=oms.error
OMS_EXECUTE_TOPIC=oms.execute
OMS_ORDER_TIMEOUT_MS=5000
OMS_RETRY_INTERVAL_MS=1000
OMS_MAX_RETRIES=3
OMS_NATS_URL=nats://localhost:4222


Alternatively, create a .env file in the project root with these values.
Running the OMS


cargo run --release
